# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | ✓ |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** open a public issue.

Instead, report it privately via GitHub's security advisory feature:
**[Report a vulnerability](../../security/advisories/new)**

We will respond within 48 hours and aim to release a fix within 7 days for critical issues.

## Security Notes for Users

### state.json — NEVER commit this file

`state.json` is generated by `xchat-bot-python` during the `login` + `unlock` flow.
It contains your **private encryption keys**. If committed to git, your XChat messages can be decrypted by anyone with access to the repo.

```bash
# Verify state.json is ignored:
git check-ignore -v state.json
# Should output: .gitignore:1:state.json  state.json

# If not ignored, add it immediately:
echo "state.json" >> .gitignore
git rm --cached state.json  # if already tracked
```

### Consumer Secret — treat like a password

Your `CONSUMER_SECRET` (in `.env`) allows anyone to forge webhook signatures.
Never commit `.env` to git. The `.gitignore` in this repo already excludes it.

### Webhook signature validation

Always validate `X-Signature-256` on incoming webhook payloads.
The playground server does this automatically when `CONSUMER_SECRET` is set.
See `playground/webhook/signature.py` for the implementation.

### X API data handling

Per [X's Developer Agreement](https://developer.x.com/en/developer-terms/agreement):
- Do not store X user data beyond what's needed for your bot's function
- Do not use X data to train AI/ML models (except as permitted by X)
- Respect user opt-out requests promptly
